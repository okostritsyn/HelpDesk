Согласно PMBOK ( Project Management Body of Knowledge) Проект - это временное предприятие, предназначенное для создания уникальных продуктов, услуг, или результатов.

1С: Документооборот 8 понимает под проектами тоже самое, правда с некоторым дополнением: Проекты также нужны для автоматизации операционной деятельности по исполнению плана проекта и для ведения аналитического учета всех данных в разрезе Проектов.

Жизненный цикл проекта

Жизненный цикл проекта – это набор, как правило, последовательных и иногда перекрывающихся фаз проекта, названия и количество которых определяются потребностями в управлении и контроле организации или организаций, вовлеченных в проект, характером самого проекта и его прикладной областью.

Жизненный цикл проекта выглядит так:

  1. Инициация проекта.
    • определение целей проекта,
    • определение внутренних и внешних заинтересованных сторон проекта,
    • выбор менеджера проекта.
  2. Планирование проекта.
    • составление задач проекта,
    • выбор исполнителей задач,
    • определение сроков выполнения.
  3. Утверждение плана проекта.
  4. Старт проектных работ.
  5. Выполнение проекта.
  6. Завершение проекта.

В начало

Создание проекта

Создание нового проекта состоит из следующих этапов:

  • создание карточки проекта,
  • добавление задач по проекту,
  • расчет трудозатрат по проекту и каждой задачи в отдельности.

Создание карточки проекта является инициацией проекта. Другими словами, в программе отмечается факт, что по какому-то направлению должен быть выполнен проект. Например: предприятие должно переехать в новый офис.

Чтобы начать создание карточки проекта, выполните следующие действия:

  1. Перейдите в раздел Совместная работа,
  2. Выполните команду Проекты панели навигации, станет доступным список проектов.
  3. Нажмите на кнопку Создать панели действий, откроется карточка нового проекта.

В карточке проекта можно сделать следующее:

  • определить плановую дату начала и окончания проекта,
  • заполнить список участников и Рабочую группу проекта,
  • определить трудозатраты по проекту.

Обязательными для заполнения являются поля:

  • окончание,
  • наименование.

Список всех действий над карточкой проекта доступен в меню Все действия.

После того, как карточка будет создана, можно согласовать проект с руководителем (команда Создать на основании, меню Все действия).

В начало

Проектная команда

Проектная команда – это список пользователей и ролей, которые выполняют работы по проекту.

Добавить пользователей в проектную команду можно в карточке проекта на закладке Проектная команда.

Важно запонить Проектную команду в самом начале работ, потому что исполнять задачи по проекту могут только участники Проектной команды.

Чтобы участники Проектной команды автоматически копировались и в Рабочую группу, нужно установить флажок Автоматически вести состав участников рабочей группы на закладке Рабочая группа.

В начало

Расписание проекта

Расписание проекта состоит из задач по проекту, которые выстроены в иерархической последовательности. По каждой задаче можно стартовать процесс для исполнения (команда Создать на основании).

Чтобы открыть расписание проекта, выполните следующие действия:

  1. Перейдите на закладку Проекты.
  2. Выберите нужный проект из списка, он станет выделен синим цветом.
  3. Перейдите на закладку Задачи проекта, станет доступно расписание проекта.

Просматривать расписание проекта можно в следующих режимах:

  • планирование,
  • исполнение,
  • трудозатраты.

Редактировать задачи можно:

  • в диалоге (добавление и редактирование в задачи в отдельной карточке),
  • в списке (добавление и изменение задачи в полях списка).

Для каждой проектной задачи нужно заполнить:

  • наименование,
  • описание,
  • плановую дату начала, окончания и длительности задачи,
  • список исполнителей проектной задачи,
  • плановые и фактические трудозатраты,
  • список предшественников.

Задачу можно также также сделать Вехой.

Веха – это точка, которая отмечает важное событие в проекте и используется для контроля за ходом выполнения проекта. Веха имеет нулевую длительность и у нее не может быть исполнителей.

Таким образом, после того, как в карточке задачи будет стоять флажок Это веха добавить исполнителей и дату окончания станет невозможно.

После сохранения в задаче автоматически проставляется код СДР.

Код СДР (структурной декомпозиции работ) – это алфавитно-цифровой код, который определяет уникальное место задачи в структуре проекта.

При помощи команд панели действия задачу можно переместить выше или ниже по списку (команды Вверх и Вниз).

К задаче можно также создать:

  • последующую задачу (команда Добавить последующую команду),
  • подчиненную задачу (команда Добавить подчиненную задачу).

Список всех действий над расписанием проекта доступен в меню Все действия командной панели.

В начало

Диаграмма Ганта

Диаграмма Ганта – это инструмент иллюстрации календарного плана. Диаграмма Ганта состоит из полос, ориентированных вдоль оси времени. Каждая полоса на диаграмме представляет отдельную задачу в составе проекта:

  • концы — моменты начала и завершения работы,
  • протяженность — длительность работы.

Вертикальной осью диаграммы служит перечень задач. Диаграмма Ганта проекта выводится на одноименной закладке. При помощи двойного щелчка мыши по полосе диаграммы можно открыть карточку задачи по проекту.

В начало

Начало проектных работ

Для организации выполнения проектных задач используются процессы. Создать процесс по задаче расписания проекта можно при помощи команды Создать на основании, которая доступна:

  • в карточке задачи проекта,
  • в командной панели на закладке Задачи проекта.

На основании проектной задачи можно создать:

  • основной процесс,
  • вспомогательный процесс.

Основной процесс – процесс, предназначенный для исполнения проектной задачи. Например: выбор дня переезда офиса.

При создании Основного процесса на основании проектной задачи в карточке процесса будут заполнены следующие поля:

  • срок,
  • проект, задача,
  • исполнитель.

При старте Основного процесса автоматически проверяется, чтобы срок исполнения процесса не превышал срок окончания проектной задачи. Если дата начала процесса меньше, чем дата начала проектной задачи, программа выдаст предупреждение при запуске. Автор должен сам сделать выбор, запускать процесс или нет.

Трудозатраты на выполнение Основного процесса будут отнесены к трудозатратам проектной задачи.

Когда по проектной задаче стартует первый Основной процесс, будет автоматически проставлено время его фактического начала в карточке проектной задачи (поле Фактическое начало).

Чтобы создать Основной процесс, выполните следующие действия:

  1. Перейдите на закладку Задачи проекта.
  2. Щелчком мыши выберите нужную задачу из списка.
  3. Выполните команду Создать на основании в панели действий.
  4. В меню доступных действий выберите Основной процесс.
  5. Заполните карточку процесса, нажмите на кнопку Записать и закрыть, чтобы стартовать процесс по задаче.

Список всех Основных процессов по проектной задаче можно посмотреть при помощи команды Процессы в карточке проектной задачи или карточке самого процесса на закладке Основные процессы.

Список Основных процессов отображается в виде дерева в порядке убывания.

Помимо Основного процесса по проекту или проектной задаче можно создать Вспомогательный процесс.

Вспомогательный процесс – это процесс, который не служит для достижения целей задачи или самого проекта. Например, процесс Ознакомление.

При старте в карточке Вспомогательного процесса будет автоматически заполнено поле Предмет.

Трудозатраты на выполнение Вспомогательного процесса не будут отнесены к трудозатратам проектной задачи.При запуске не будет проверки соответствия общему сроку проектной задачи.

Чтобы создать Вспомогательный процесс по проектной задаче, выполните следующие действия:

  1. Перейдите на закладку Проектные задачи.
  2. Щелчком мыши выберите нужную задачу из списка.
  3. Выполните команду Создать на основании в панели действий.
  4. В меню доступных действий выберите Вспомогательный процесс.
  5. Заполните карточку процесса, нажмите на кнопку Записать и закрыть, чтобы стартовать процесс по задаче.

Список всех Вспомогательных процессов по проектной задаче можно посмотреть при помощи команды Процессы в карточке проектной задачи на закладке Вспомогательные процессы или карточке самого процесса также на закладке Вспомогательные процессы. Список Вспомогательных процессов также отображаются в виде дерева в порядке убывания.

Выполнение задач по проектам и учет трудозатрат

После старта процесса на основании проектной задачи, исполняющее лицо получит обычную задачу по процессу, типа Ознакомление, Исполнение, Согласование и так далее.В карточке задачи будет доступна ссылка на карточку проекта в поле Проект.

При выполнении задачи исполняющее лицо должно указать фактические трудозатраты по задаче.

Трудозатраты – это расход труда сотрудников на выполнение какого-либо вида работы. Обычно выражается в рабочих человеко-часах, в человеко-днях или рабочих неделях.

После того как администратор включит учет трудозатрат в Настройках программы, ежедневный отчет будет автоматически заполняться данными о трудозатратах из карточек задач, документов, файлов, писем и других.

Вести трудозатраты из карточки задачи можно несколькими способами:

  • команда Включить хронометраж панели действий (при нажатии на кнопку завершения работы над задачей, например Исполнено, программа предложит сделать запись о трудозатратах);
  • команда Указать трудозатраты панели действий (после выполнения команды можно вручную добавить трудозатраты по задаче без использования хронометража);
  • команда Трудозатраты панели навигации (можно также вручную добавить трудозатраты по задаче без использования хронометража ).

После нажатия на кнопку Добавить, трудозатраты автоматически копируются в Ежедневный отчет пользователя.

Контроль сроков исполнения

Поскольку исполнение проектов выполняется при помощи процессов, контроль проектных задач осуществляется на нескольких уровнях:

  • на уровне задач процессов при помощи отчетов по задачам,
  • на уровне процессов в целом (отчет Процессы показывает просроченные процессы с возможностью отбора по проектам),
  • на уровне проектных задач (отчет Проектные задачи показывает просроченные проектные задачи).

Кроме того есть возможность контролировать исполнение задач непосредственно в расписании проекта на закладке Задачи проекта. При этом в поле Режим просмотра на панели действия должно стоять Контроль выполнения.

Если плановая дата проектной задачи наступила, но фактическое выполнение еще не началось, она будут отмечена красным в колонке Начало план.Если плановый срок задачи просрочен, она будет отмечена красным в колонке Окончание план.

    Анализ проектов

    Анализировать и контролировать проекты можно при помощи отчета Проекты.

    Разные варианты отчета дают возможность узнать ответы на следующие вопросы:

    • сколько проектов было завершено за период,
    • какие проекты еще в работе,
    • кто из руководителей и какие проекты вел или ведет,
    • насколько велика разница между плановыми и фактическими трудозатратами по проектам.

    Сформировать отчет Проекты можно при помощи команды Проекты панели действий раздела Совместная работа.

    Для удобства анализа текущего проекта на закладке Задачи проекта доступна команда Отчеты. Отчет, сформированный таким способом, будет содержать информацию только по текущему проекту.

    Доступны следующие варианты отчета по проекту:

    • текущее состояние задач,
    • скоро должны завершиться,
    • скоро должны начаться,
    • план-факт по срокам (показывает, есть ли превышение плановых сроков),
    • план-факт по трудозатратам (показывает плановые и фактические трудозатраты по проекту.

    Загрузка проектов из Microsoft Project

    В программе есть возможность загрузить проект из Microsoft Project. Это может быть полезно, для организации исолнения проектов, подготовленных в Microsoft Project. После составления проект можно загрузить его в 1С:Документооброт и начать работу.

    Загрузить проект можно при помощи команды Загрузить из Microsoft Project, которая доступна:

    • в командной панели на закладке Проекты,
    • в командной панели на закладке Задачи проекта.

    В программу можно загрузить файл проекта Microsoft Project с расширением XML. При загрузке не переносятся ресурсы (исполнители) задач и календари проекта, проектных задач и ресурсов. Эти данные необходимо настроить самостоятельно перед началом загрузки.

    Чтобы загрузить проект из Microsoft Project, выполните следующие действия:

    1. Перейдите в раздел Совместная работа.
    2. Выполните команду Проекты панели навигации, откроется список проектов.
    3. Выполните команду Загрузить из Microsoft Project.
    4. В помощнике загрузки проекта выберите файл, из которого будет производится загрузка.
    5. Выберите проект, в который будет происходить загрузка. Если хотите создать новый проект, очистите поле Проект.
    6. Нажмите на кнопку Загрузить, чтобы начать загрузку проекта в программу.

    В программе предусмотрена возможность загрузки проектов только из Microsoft Project. Загрузить проект из 1С: Документооборот в Microsoft Project нельзя.

    Права доступа и рабочая группа

    Права на чтение проектов имеют все пользователи. Права ограничиваются при помощи грифа доступа, организации и рабочей группы.

    Права на создание проектов имеют только пользователи,которые входят в группу доступа Руководители проектов.

    Изменять проекты может только пользователь, указанный в карточке проекта как руководитель.

    Создавать и изменять проектные задачи может только руководитель проекта.

    Только руководитель проекта может видеть все фактические трудозатраты по проектам.

    Рабочая группа – это список пользователей и ролей, у которых есть права для просмотра проекта. Заполнить Рабочую группу можно на закладке Рабочая группа карточки проекта.

    Поскольку, права на просмотр есть у всех пользователей, заполнять Рабочую группу желательно, только в том случае, если нужно ограничить права на просмотр проекта.

    Советы

    Чтобы изменить длительность задачи при помощи диаграммы Ганта, растяните полосу задачи по краям вдоль оси времени.

    Можно перемещать вних и вверх несколько выделенных проектных задач.

    Чтобы видеть только нужные аттрибуты задач, используйте режимы просмотра.

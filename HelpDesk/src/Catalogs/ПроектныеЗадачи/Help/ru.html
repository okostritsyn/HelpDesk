Справочник Проектные задачи содержит описание всех работ, выполняемых по проекту, в виде иерархической структуры.

Для каждой проектной задачи необходимо заполнить:

  • наименование,
  • описание,
  • плановую дату начала, окончания и длительности задачи,
  • список исполнителей проектной задачи,
  • плановые трудозатраты исполнителей,
  • список предшественников,
  • график работы (если не указан, то будет использован график проекта).


В качестве исполнителей можно указать пользователя, роль исполнителя, корреспондента, контактное лицо корреспондента. Если в проектной задачи указаны исполнители, то общие плановые трудозатраты рассчитываются как сумма плановых трудозатрат исполнителей.

Задачу можно сделать Вехой. Веха – это точка, которая отмечает важное событие в проекте и используется для контроля за ходом выполнения проекта. Веха имеет нулевую длительность и у нее не может быть исполнителей.
После сохранения в задаче автоматически проставляется код СДР. Код СДР (структурной декомпозиции работ) – это алфавитно-цифровой код, который определяет уникальное место задачи в структуре проекта.

Плановые даты проектных задач рассчитываются автоматически исходя из длительности задачи, указанных предшественников и указанных ограничений. При указании предшественников можно указать следующие типы зависимости:

  • начало-окончание,
  • начало-начало,
  • окончание-начало,
  • окончание-окончание.

Также можно указать задержку по отношению к предшественнику. При расчете плана учитываются указанные предшественники для самой задачи, и для всех ее родительских задач.

В проектной задаче можно указать Тип ограничения, который ограничивает допустимую дату начала или завершения задачи. Используются следующие типы ограничений:

  • как можно позже,
  • как можно раньше,
  • окончание не ранее,
  • окончание не позднее,
  • начало не ранее,
  • начало не позднее,
  • фиксированное начало,
  • фиксированное окончание.

Если выбрано ограничение, отличное от Как можно позже или Как можно раньше, следует ввести соответствующую дату в поле Дата ограничения.

Для организации выполнения проектных задач используются процессы. Создать процесс по задаче проекта можно при помощи команды Создать на основании, которая доступна:

  • в карточке задачи проекта,
  • в командной панели плана проекта на закладке Задачи проекта.


На основании проектной задачи можно создать:

  • основной процесс (используется для исполнения проектной задачи),
  • вспомогательный процесс (не относится непосредственно к исполнению проектной задачи).


Основной процесс – процесс, предназначенный для исполнения проектной задачи. При создании основного процесса на основании проектной задачи в карточке процесса будут заполнены следующие поля:

  • срок,
  • проект, задача,
  • плановые трудозатраты,
  • исполнители.


Когда по проектной задаче стартует первый основной процесс, будет автоматически проставлено время его фактического начала в карточке проектной задачи (поле Фактическое начало). Когда по проектной задаче завершается последний основной процесс, будет автоматически проставлено время его фактического окончания в карточке проектной задачи (поле Фактическое окончание). Трудозатраты на выполнение основного процесса будут отнесены к трудозатратам проектной задачи.

Список всех основных процессов по проектной задаче можно посмотреть при помощи команды Процессы в карточке проектной задачи на закладке Основные процессы.

Помимо основного процесса по проекту или проектной задаче можно создать Вспомогательный процесс. 
Вспомогательный процесс – это процесс, который не служит для достижения целей задачи или самого проекта.
При старте в карточке вспомогательного процесса будет автоматически заполнено поле Предмет. Поле Проект, задача заполнено не будет. Трудозатраты на выполнение вспомогательного процесса не будут отнесены к трудозатратам проектной задачи. Дата начала и окончания вспомогательного процесса не влияет на фактическое начало и окончание проектной задачи.

Список всех вспомогательных процессов по проектной задаче можно посмотреть при помощи команды Процессы в карточке проектной задачи на закладке Вспомогательные процессы.
